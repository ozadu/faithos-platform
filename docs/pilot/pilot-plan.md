# Pilot Plan

## Audience

- Church administrators
- Office secretaries
- Department heads
- Technical installer/admin

## Suggested sequence

1. Install FaithOS and verify Docker health.
2. Create the first administrator.
3. Complete the organization profile.
4. Create departments, users, roles, and workflow assignments.
5. Disable demo seed and hide demo credentials for production-like pilot use.
6. Run the UAT test script.
7. Run backup and restore dry-runs.
8. Start a limited live pilot with selected departments.
9. Review feedback daily.
10. Decide whether to extend, pause, or prepare a wider release.

Keep the first pilot small enough that staff can ask questions and the technical
admin can recover quickly if configuration needs correction.
