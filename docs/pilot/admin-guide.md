# Pilot Admin Guide

Administrators are responsible for:

- Creating the first admin at `/setup` when no admin exists.
- Disabling demo seed and demo credential display in production-like
  environments.
- Managing users, departments, roles, permissions, document types, and workflow
  assignments.
- Reviewing health, audit log, reports, feedback, and pilot issues.
- Running backup and restore drills.

Use Swagger at `/api/docs` for endpoint inspection, but perform UAT through the
web application wherever possible.
