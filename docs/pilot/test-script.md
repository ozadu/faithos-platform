# Pilot Test Script

Use `/pilot/uat` to record pass/fail results.

Minimum flows:

- Login
- First-admin setup
- Organization profile
- Departments
- Users and roles
- Document creation
- Attachment upload/download
- Document submission
- Document routing
- Workflow start, approve, return, reject, complete
- Notifications
- Dashboards
- Reports and CSV export
- Feedback submission and admin review
- Backup dry-run
- Restore dry-run

Every failed test should include notes, expected result, actual result, and owner.
