# Feedback and Issue Workflow

Staff feedback:

1. Staff submit feedback at `/feedback`.
2. Admins review feedback at `/admin/feedback`.
3. Admins mark feedback `REVIEWED` or `RESOLVED`.
4. Admins create related issue tracker items at `/admin/pilot-issues`.

Issue tracker:

- `Open`: issue captured.
- `In Review`: owner is investigating.
- `Planned`: fix or operational action is scheduled.
- `Fixed`: implementation or operational fix completed.
- `Closed`: verified and accepted.

The issue tracker is lightweight pilot triage, not a replacement for a full
external support desk.
