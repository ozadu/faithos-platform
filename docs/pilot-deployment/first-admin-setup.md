# First Admin Setup

The first administrator owns initial configuration and must be a trusted pilot operator.

## Steps

1. Start the FaithOS stack.
2. Open `/setup`.
3. Confirm the setup page reports that first-admin setup is available.
4. Create the pilot organization and first admin account.
5. Use a real admin email address.
6. Use a strong temporary password and change it after handover if required.
7. Log in at `/login`.
8. Confirm `/setup` is disabled after the first active administrator exists.

## Expected result

- First organization exists.
- First admin user exists.
- Admin has the required admin permissions.
- Setup cannot be repeated accidentally.

## Evidence to capture

- Date/time setup was completed.
- Admin email used.
- Confirmation that `/setup` is disabled afterward.

Do not share the first admin password in reports or screenshots.
