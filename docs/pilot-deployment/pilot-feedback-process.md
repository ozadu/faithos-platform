# Pilot Feedback Process

Feedback is intentionally lightweight in v0.9.0. It is not a full helpdesk.

## Staff process

1. Open `/feedback`.
2. Choose category and severity.
3. Describe what happened and what was expected.
4. Add screenshot link if available.
5. Submit feedback.

## Admin process

1. Open `/admin/feedback`.
2. Filter by category, severity, or status.
3. Review new feedback daily during the pilot.
4. Update status:
   - `NEW`
   - `REVIEWED`
   - `PLANNED`
   - `RESOLVED`
   - `WONT_FIX`
5. Copy true defects into the pilot issue log.
6. Summarize patterns in the final pilot report.

## Response targets

- Critical: same day
- High: within two business days
- Medium: weekly review
- Low: backlog review before v1.0.0 planning
