# User Onboarding

Use `/admin/users`, `/admin/users/import`, and `/admin/users/onboarding` to prepare pilot staff.

## Minimum user data

- First name
- Last name
- Valid email
- Department
- Role
- Active status

## Steps

1. Create or import only users participating in the pilot.
2. Assign every active user to a department.
3. Assign every active user to a role.
4. Confirm each user has a valid email address.
5. Send temporary passwords through the configured development email flow when appropriate.
6. Ask each user to log in once before the pilot starts.
7. Review `/admin/users/onboarding` for:
   - Users without role
   - Users without department
   - Inactive users
   - Demo/default password risk
   - Users who have not logged in

## Expected result

Every pilot user is active, assigned to a department, assigned to a role, has a valid email, and can log in.
