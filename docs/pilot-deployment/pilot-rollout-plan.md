# Pilot Rollout Plan

## Phase 1: Technical preparation

- Confirm host machine and Docker readiness.
- Configure `.env`.
- Start stack and verify services.
- Complete backup and restore dry-runs.

## Phase 2: Administrative setup

- Create first admin.
- Complete organization profile.
- Create departments.
- Create users and roles.
- Review permissions.
- Configure workflow templates and assignments.

## Phase 3: Training and UAT

- Train administrators.
- Train staff users.
- Run `/pilot/uat`.
- Record pass/fail notes and issues.
- Fix or accept issues before live pilot.

## Phase 4: Controlled live pilot

- Select a small set of real document flows.
- Run the pilot for one to two weeks.
- Review feedback and issues daily.
- Keep paper fallback available if the pilot owner requires it.

## Phase 5: Pilot report

- Count users, departments, documents, workflows, issues, and feedback.
- Summarize technical, security, and operational observations.
- Recommend go, pause, or no-go for wider rollout.
